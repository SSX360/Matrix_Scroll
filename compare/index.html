Honest positioning

How Matrix Scroll compares

Matrix Scroll signs Git commit envelopes: portable attestations of who produced a change, attached to the commit itself. Most adjacent tools are complements, not substitutes.

One line: Sigstore and SLSA answer "what did CI build?" Matrix Scroll answers "who attested this commit before it was pushed?"

Commit-time

Matrix Scroll focuses on signatures attached before push, not only on metadata created after CI builds artifacts.

Offline

The verifier path stays local in the CLI or browser instead of assuming a connected trust service.

Open schema

The standard JSON envelope format is designed to be easily parsed and verified by any client.

Closest commit-level peers

agentmark

Similar: Commit-time Ed25519 manifests and a CI verification gate.

Different: Primarily software pipeline keys today. Matrix Scroll offers a structured, open-source schema with standardized verification tools.

Alien Agent ID

Similar: Ed25519 commit signing and public verification.

Different: Identity binding through cloud OIDC / DPoP. Matrix Scroll emphasizes offline verification and a transparent offline signature contract.

ForgeProof

Similar: Provenance for AI-assisted code.

Different: File-level receipts before build, not commit envelopes that travel with every clone.

Established supply chain (CI / artifact)

Sigstore, GitHub attestations, SLSA, in-toto, and GUAC secure build outputs. They do not record which AI tool authored a line at commit time. Matrix Scroll envelopes can feed those systems instead of competing with them.

What Matrix Scroll combines

  1. Commit-time signing instead of only post-build artifact attestation
  2. Agent and human actor data in the envelope schema
  3. An emulated developer mode with standardized verification
  4. Offline verification in the CLI or browser

Honest gaps we are still building

  • More public adopter proof
  • Multi-agent commits in one envelope
  • Rekor / transparency log integration
