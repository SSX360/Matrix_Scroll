Authorized Payment Agent (AP2)

The Cryptographic Trust Layer for Autonomous Agent Payments

Preventing agent transaction hijacking and establishing human-to-agent delegation rules. Designed for compatibility with 60+ payment networks and crypto rails.

Why Agent Payments Need a Trust Layer

When an AI agent initiates a transaction via a virtual credit card or a cryptocurrency wallet, traditional banking rails only see the card token or wallet key. They cannot distinguish between an authorized purchase ordered by the human owner and a hijacked transaction executed by a rogue agent or prompt injection. Matrix Scroll provides the cryptographic attestation linking the specific payment intent to the owner's identity certificate.

1. The Delegation & Trust Chain

The AP2 specification operates on a strict chain-of-custody model:

  1. Identity Certificate (Scroll Pass): The human owner signs their agent's public key, binding it to a verified profile (e.g. GitHub, Google) via the Hosted Identity Authority.
  2. Agent Signature: The agent signs the specific transaction intent (specifying transaction ID, merchant, amount, currency, and payment card token hash).
  3. Verification: The merchant or payment processor fetches the trust root and verifies that the agent signature is cryptographically valid and chained to an authorized owner.

2. Network Compatibility

Our standardized open wire format is designed for compatibility across 60+ payment networks and crypto institutions. AP2 attestation envelopes are network-agnostic — merchants and processors verify the cryptographic chain, not a proprietary rail.

3. Attestation JSON Schema

Every AP2 transaction attestation is packaged as a standard `matrixscroll.agent_payment.v1` envelope: 

{
  "schema": "matrixscroll.agent_payment.v1",
  "transaction_id": "tx_ok12345",
  "amount": 250.0,
  "currency": "USD",
  "merchant": "OpenAI",
  "payment_method": {
    "type": "virtual_card",
    "identifier_hash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b..."
  },
  "agent_device_id": "MS-4319-20D5",
  "timestamp": "2026-06-27T04:29:41Z",
  "signature": {
    "schema": "matrixscroll.signature.v1",
    "algorithm": "ed25519",
    "device_id": "MS-4319-20D5",
    "public_key": "bsVoWgUYK+NZNcSNIoWj...",
    "value": "+dRjYqhW+Bf/1L7Ka0jZz..."
  }
}
