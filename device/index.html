Hardware preview

Software first. Preview trust upgrade next.

Matrix Scroll ships as software today. The SSX360 SE050 lane is a design-partner preview for teams that want non-exportable Ed25519 signing without changing the commit envelope schema or the verifier used in the CLI, browser, and CI.

Shipping now: PyPI 0.2.6, Git hooks, Scroll Gate PR verification, browser verifier, and emulated-mode evaluation.
In progress: SSX360 SE050 hardware preview and verifier-compatible external Ed25519 signer guidance.
Not: IAM, sandboxing, prompt filtering, or an agent runtime.

Scroll Key preview

A USB-C secure-element reference path built to keep the private key inside the device while signing the same canonical manifest bytes Matrix Scroll already uses today.

Scroll Token concept

A future packaging concept for teams that want a portable, tangible trust anchor. The public launch story stays software-first until the acceptance package is complete.

Why this path exists

Same verifier contract

The hardware path does not introduce a new schema, algorithm, or review flow. It signs the same canonical bytes and lands on the same proof surfaces.

Host cannot export the key

The goal is to keep the private key sealed inside the secure element so local compromise does not turn into silent key exfiltration.

Design-partner only

Preview access is intentionally narrow until latency, provisioning, and acceptance criteria are all documented against the existing verifier contract.
