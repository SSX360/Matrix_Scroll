SPEC.md
Protocol surface

Pure Ed25519 over canonical JSON bytes.

Matrix Scroll v1 keeps the trust contract narrow: canonicalize the manifest, remove the top-level signature block, derive a human-readable device ID from the public key, and verify the Ed25519 signature over the resulting bytes.

Shipping now: PyPI 0.2.6, Git hooks, Scroll Gate PR verification, browser verifier, and emulated-mode evaluation.
In progress: SSX360 SE050 hardware preview and verifier-compatible external Ed25519 signer guidance.
Not: IAM, sandboxing, prompt filtering, or an agent runtime.

Algorithm

Matrix Scroll v1 signs canonical manifest bytes with Ed25519 only. Any other signature algorithm is non-conforming.

Device ID

The verifier derives MS-XXXX-XXXX from the SHA-256 digest of the raw public key bytes. The public key is the real identity; the device ID is a human-readable handle.

Canonical bytes

Remove the top-level signature block, sort keys recursively, ASCII-escape non-ASCII characters, reject NaN or Infinity, and UTF-8 encode before verification.

Reference canonicalization

body = {k: v for k, v in payload.items() if k != "signature"}
json.dumps(
    body,
    sort_keys=True,
    ensure_ascii=True,
    allow_nan=False,
    separators=(",", ":"),
).encode("utf-8")
