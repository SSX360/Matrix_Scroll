Hardware Architecture

Technical Specifications

Designed as a dedicated Matrix Scroll signing backend. The preview path keeps pure Ed25519 over canonical manifest bytes and avoids introducing a second verifier model.

PARAMETER
HARDWARE REGISTER / SPECIFICATION SUMMARY
[SECURE_ELEMENT]
NXP SE050 (CC EAL6+ Certified) Mints and stores the cryptographic identity key pairs (Ed25519) in isolated, tamperproof hardware. The private key cannot be read, copied, or extracted by the host operating system or the local microcontroller.
[CONTROLLER_MCU]
Raspberry Pi RP2350 microcontroller Drives the 1-bit Sharp Memory display, processes capacitive touch, and manages the USB-C serial interface. Crucially, the MCU acts as a gateway and cannot inspect or retrieve private key material from the secure element.
[AVATAR_DISPLAY]
1.3" Sharp Memory LCD (concept form factor only) Optional concept display for the desk form factor. It is not required for the core SE050 signing path or the public verification model.
[PRESENCE_SENSOR]
Capacitive touch-to-sign pad Optional human-gated signing control for the concept token form factor. The core Scroll Key preview does not depend on this UX layer.
[BUS_INTERFACE]
USB CDC ACM transport Documented newline-delimited JSON over USB with ping, pubkey, and sign for the preview path. The RP2350 handles transport and local UX while the SE050 keeps the signing key non-exportable.
[CHASSIS_ENCLOSURE]
CNC Milled Matte Black Aluminum Keyring-sized matte anodized aluminum shell with a signature lime (#e8ff9c) accent strip. Heavy, tactile, premium desk object aesthetic.

SE050 hardware roadmap

One verify path

CLI, browser, and CI all verify the same canonical manifest bytes, public key, device ID, and signature block. Reviewers do not need a hosted control plane.

Canonical JSON signatures

Sorted keys, compact separators, ASCII escaping, and signature-block exclusion keep the signing input stable across implementations and audits.

Offline proof surfaces

Audit-ready evidence

Commit envelopes map cleanly onto reviewer, compliance, and supply-chain workflows. Matrix Scroll complements GHAS, Semgrep, Sigstore, SLSA, and existing hardware roots instead of replacing them.

How it works

From install to offline proof.

01

Install the SDK

Install Matrix Scroll in one repo, enable the Git hook, and start with the shipping emulated Ed25519 identity used for commit envelopes today.

02

Sign the commit

The hook canonicalizes the manifest bytes, signs them with the active provider, and attaches a commit envelope that records actor, tool, and scope.

03

Verify anywhere

Reviewers verify the same envelope in the CLI, browser, or CI. When the SE050 path ships, it plugs into this flow without changing the verifier or schema.
