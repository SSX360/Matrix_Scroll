06 Common questions

What is Matrix Scroll and how does it secure Git?

Matrix Scroll is signed commit-time provenance for agent-assisted Git. It attaches an Ed25519-signed commit envelope that records who or what signed a change, which tool produced it, and optional bounded scope, then lets reviewers verify that proof offline in the CLI, browser, or CI before merge.

How do hardware and emulated modes differ?

Emulated mode ships today and stores the signing key locally so teams can evaluate the full flow now. Hardware mode keeps the same envelope schema and verifier contract, but moves the private key into the SE050 secure element; that path remains preview-only.

How do I integrate Matrix Scroll into CI/CD?

Install hooks, publish envelopes to refs/notes/matrixscroll, then run SSX360/matrixscroll-verify-action@v1 on the PR commit range. Protected branches can require Matrix Scroll proof alongside existing scanners and branch protection.
