Security posture

A root of trust you can audit.

Every capability is designed so trust originates in hardware and follows the work across editors.

Ed25519 Cryptographic Identity

Cryptographic proof of origin. Every code write is signed directly by the device key. This cryptographically attributes each change to a paired device, eliminating key spoofing and identity theft.

Isolated Hardware Key Storage

NXP SE050 Secure Element. Private signing keys are generated inside dedicated secure silicon and can never be extracted or copied. Even if a developer's laptop is compromised by malware, your production keys remain safe.

Cross-IDE Context Protocol

Universal editor support (MCP). Seamless integration across Cursor, VS Code, and JetBrains. Security and attribution policies are enforced uniformly, no matter which IDE your developers prefer.

BM25 Local Context Ranking

Hallucination-free indexing. Our local relevance engine filters codebase context before it goes to the AI. This cuts context sent to the model and guarantees the AI only edits code using highly relevant local context.

Air-Gapped LLM Failover

Offline security compliance. Automatically falls back from cloud APIs to local models (like Ollama) when handling ultra-sensitive code or working offline. Zero source code leakage to third parties.

Attested Supply Chain Evidence

Audit-ready compliance. Generates tamperproof, hardware-attested evidence for every single AI modification. This produces hardware-attested evidence that maps onto SOC 2 CC8 / SLSA provenance controls.
