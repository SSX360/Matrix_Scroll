[02] Process

Tap to approve. Nothing more.

01

Agent requests payment

Your AI agent needs to authorize a transaction — a SaaS subscription, cloud compute, or API call.

02

Phone prompts for tap

The companion app displays mandate details: amount, vendor, and policy context. It asks for physical approval.

03

Card signs on-chip

You tap the AP2 Vault Card. The NXP SE050 (CC EAL6+) secure element signs with Ed25519. The private key never leaves the chip.

04

Payment executes

The signed envelope is verified by the SSX360 control plane. The transaction proceeds — cryptographically proven to be human-approved.

[03] Architecture

Hardware-rooted trust.

SECURE ELEMENT

NXP SE050 (CC EAL6+) — tamper-resistant chip certification. Holds the Ed25519 private key in hardware; certification applies to the secure element, not the whole card assembly.

NFC INTERFACE

Passive operation — no battery. The antenna powers the chip and carries the signed payload in a single tap.

CONTROL PLANE

SSX360 verifies signatures, enforces spend policies, and provides real-time audit trails for every approval.

[04] Security model

Why hardware beats software.

01

Key never extracted

The SE050 generates and stores the private key internally. There is no API to export it.

02

No battery, no attack surface

Passive NFC means no Bluetooth, no WiFi, no firmware update channel. The card is air-gapped by physics.

03

Possession factor

Software can be copied. This card cannot. Physical presence is required for every approval.

04

Tamper-evident silicon

CC EAL6+ applies to the NXP SE050 secure element — the chip resists physical intrusion at certified levels.

05

Cryptographic proof

Every approval produces a verifiable Ed25519 signature. The audit trail is mathematically checkable offline.

06

Policy enforcement

SSX360 enforces spend limits, vendor allowlists, and time-of-day restrictions before the card is prompted.
