Browser verifier

Verify signed commit envelopes in your browser.

Matrix Scroll uses the same public Ed25519 contract in the CLI, browser, and CI. Paste an envelope, load a known-good sample, or tamper it to watch verification fail loud.

Private by default: verification runs entirely in your browser. Envelopes you paste or edit here are not uploaded to Matrix Scroll servers.

Shipping now: PyPI 0.2.6, Git hooks, Scroll Gate PR verification, browser verifier, and emulated-mode evaluation.
In progress: SSX360 SE050 hardware preview and verifier-compatible external Ed25519 signer guidance.
Not: IAM, sandboxing, prompt filtering, or an agent runtime.

What this checks

The verifier removes the top-level signature block, canonicalizes the remaining JSON, derives the expected device_id from the public key, and runs Ed25519 verification over the canonical bytes.

Proof surfaces that stay aligned

Same contract everywhere

The browser verifier checks the same canonical bytes, public key, device ID, mode, and signature block that the SDK and Scroll Gate check in CI.

Fails loud on tampering

Device ID mismatches, invalid base64, unsupported algorithms, or broken signatures all surface directly in the verifier so reviewers can see why a manifest does not pass.

Software-first today

matrixscroll==0.2.6 and SSX360/matrixscroll-verify-action@v1 stay pinned for the launch path. MATRIXSCROLL_MODE=emulated is the launch path, and hardware is a preview trust upgrade rather than a prerequisite to try Matrix Scroll.
